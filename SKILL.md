# 简历PDF生成器

## 技能概述
本技能帮助用户将零散的个人经历和项目信息，整理成符合招聘标准的专业简历，并最终生成PDF格式文件。核心特点是**严格基于用户提供的真实信息**，结合目标岗位的职位描述(JD)进行针对性优化，绝不编造任何经历或技能。

## 脚本路径说明

以下脚本路径均相对于 skill base directory（skill 所在目录），不依赖当前工作目录：

| 操作 | 命令 |
|------|------|
| 安装依赖 | `python scripts/install_deps.py` |
| 生成 PDF | `python scripts/make_resume.py path/to/resume.typ` |

这会将 Typst 编译器下载到 `assets/bin/`。

---

## 数据存储

用户数据统一存放在工作空间的 `user_data/` 目录下，Skill 本身不存储用户数据：

- `user_data/profile.md` — 基本信息、教育背景、工作经历、技能证书
- `user_data/projects/` — 项目素材库，每个项目一个 `.md` 文件

这样做的好处：Skill 可独立分发升级，用户数据不受影响。

## 启动流程

### 检测用户数据

激活 Skill 时，**首先检查** `user_data/profile.md` 是否存在：

| 状态 | 行为 |
|------|------|
| 不存在 | 进入 **Onboarding 流程**，引导创建素材库 |
| 存在 | 进入 **主流程**，读取已有数据，按需更新 |

### Onboarding 流程（首次使用）

#### 步骤一：引导用户提供素材

依次引导用户提供以下内容（可分多次完成）：

1. **基本信息**：姓名、联系电话、邮箱、求职意向、城市
2. **教育背景**：本科/硕士/博士的学校、专业、时间
3. **工作经历**：公司、部门、职位、时间、主要职责
4. **项目经历**：项目名称、时间、角色、STAR 原始素材（可稍后补充）
5. **技能与证书**：专业技能、语言能力、资质证书

#### 步骤二：保存到 user_data/

- `user_data/profile.md` → 个人信息、教育背景、工作经历、技能证书
- `user_data/projects/项目名.md` → 每个项目的 STAR 原始素材

#### 步骤三：完成确认

告知用户素材库已创建，之后激活时无需重复输入。如需新增或更新项目，随时说"录入新项目"或"更新 XXX"。

## 主流程：信息收集与验证

**原则**：已存在于 `user_data/` 的数据直接复用，**只要求用户补充缺失或过时的部分**，不要重复询问已提供的信息。

### 信息收集规则

| 数据类型 | user_data/ 中已有 | user_data/ 中没有 |
|----------|------------------|------------------|
| 基本信息 | 询问是否需要更新 | 正常收集 |
| 教育背景 | 询问是否需要更新 | 正常收集 |
| 工作经历 | 询问是否有变化 | 正常收集 |
| 项目经历 | 询问新增 or 更新 | 正常收集 |
| 技能证书 | 询问是否需要更新 | 正常收集 |
| 目标 JD | — | 必须提供 |

### 必须收集的信息清单
1. **个人基本信息**
   - 姓名
   - 联系电话
   - 电子邮箱
   - 可选：求职意向(明确的岗位名称)、所在城市、GitHub链接、个人作品集链接
2. **教育背景**
   - 个人教育阶段：本科、硕士研究生、博士研究生
   - 各教育阶段的学校名称、专业、入学及毕业时间（如果一个人是硕士研究生学历，那么需要提供本科以及硕士研究生 2个阶段；如果是博士研究生，需要询问是否存在硕士研究生经历，如果存在，需要提供本科硕士博士三个阶段的信息，如果不存在，需要提供本科博士两个阶段的信息）
   - 可选：GPA(3.5/4.0以上推荐)、相关课程、荣誉奖项
3. **工作经历**(按时间倒序，最近3-5年为主)
   - 公司名称、部门、职位、任职时间
   - 主要工作职责
   - 关键成就和贡献(尽可能提供数字)
4. **项目经历**(核心部分)
   - 项目名称、时间、担任角色
   - 项目背景和目标
   - 你具体负责的工作和采取的行动
   - 项目最终成果和影响(必须量化)
5. **技能与证书**
   - 专业技能(如编程语言、框架、工具)
   - 语言能力
   - 相关证书和资质
6. **目标岗位JD**
   - 完整的职位描述文本
   - 这是简历针对性优化的关键依据，**必须提供**

### 岗位定制化指引分支
在开始生成简历前，首先根据目标岗位判断是否需要加载专属指引，**仅匹配岗位时才读取对应文件，其他情况不读取任何额外岗位文件**，严格按照文件中的专属指引进行简历优化：
1. 如目标岗位为 **虚幻游戏引擎工程师**，读取参考文件 `references/unreal-engine-engineer.md`
2. 如目标岗位为 **金融工程产品经理**，读取参考文件 `references/quant-finance-pm.md`
3. 如目标岗位为 **推荐系统工程师**，读取参考文件 `references/recommendation-system-engineer.md`
4. 其他所有岗位：
   - 不读取任何额外的岗位专属文件
   - 直接使用下方的**默认社招简历标准**进行处理

### 默认社招简历标准
严格按照以下顺序组织内容：
1. 个人信息与求职意向
2. 核心能力摘要(可选)：3-5句话高度概括核心竞争力
3. 工作经历(核心模块)：按时间倒序
4. 项目经历(核心模块，大量篇幅)：按重要性或时间倒序
5. 教育背景
6. 技能与证书

### 经历描述标准：STAR法则
所有工作和项目经历必须使用**STAR法则**进行结构化描述：
- **S(情境)**：简要说明项目背景和面临的挑战
- **T(任务)**：明确你在项目中承担的具体职责
- **A(行动)**：详细描述你采取的具体行动和使用的方法
- **R(结果)**：**必须用数字量化成果**，展示你的贡献和影响力

**示例**：
- ❌ 错误：负责用户增长工作
- ✅ 正确：主导用户增长项目(S)，负责提升新用户注册转化率(T)，通过优化注册流程和引入社交裂变机制(A)，3个月内将注册转化率从5.2%提升至18.6%，日均新增用户增长137%(R)

### JD匹配优化原则
1. 仔细分析目标岗位JD，提取关键词和核心能力要求
2. 将用户的经历与这些要求进行匹配，调整表述顺序和侧重点
3. 在不改变事实的前提下，使用JD中的行业术语和表述方式
4. 突出与岗位最相关的经历和技能，弱化不相关内容
5. 确保简历中出现JD中80%以上的核心关键词(ATS系统友好)

### 重要禁忌
- ❌ 绝对禁止编造任何经历、技能或成果
- ❌ 禁止夸大其词，所有数据必须基于用户提供的信息
- ❌ 禁止使用模糊空洞的表述，如"具有很强的沟通能力"
- ❌ 禁止出现与求职岗位无关的个人信息，如年龄、婚姻状况、政治面貌等

## 项目录入流程

**触发词**：「录入新项目」「添加一个项目」「我有个新项目」

### 步骤一：收集项目基本信息
- 项目名称、时间周期、担任角色、团队规模

### 步骤二：收集 STAR 原始素材
- S（情境）：项目背景和面临的挑战
- T（任务）：你的具体职责和目标
- A（行动）：你具体做了什么（不是团队整体）
- R（结果）：量化成果（尽量有数字）

### 步骤三：保存到 user_data/
- 文件路径：`user_data/projects/项目名.md`
- 格式参考项目模板

### 步骤四：确认
告知用户项目已保存，可随时用于简历生成。

## 第三步：生成TYP格式简历

> ⚠️ **强制前置：生成 .typ 文件前，必须先读取 `references/typst-caveats.md`，逐项检查避免编译错误。**

**typst-caveats.md 核心检查清单**（生成文件前必查）：
- [ ] `C++` 改为 `C\+\+`
- [ ] `C#` 改为 `C\#`
- [ ] list item 中的括号改为逗号或空格
- [ ] `project.with()` 中必须包含 `author: (name: "姓名")` 参数
- [ ] 文件保存为 UTF-8 无 BOM 编码

**数据来源**：优先从 `user_data/` 读取 profile.md 和 projects/；如果没有持久化数据，则要求用户输入。

**处理逻辑**：
- 分析 JD 关键词，从 projects/ 中挑选 2-4 个最匹配的项目
- 按 STAR 法则优化每个项目的描述
- 按 JD 语言调整措辞和侧重点

如果已经提供 TYP 格式简历，则将整理优化后的内容，严格按照 `resume-ng` 模板格式，基于现有简历进行优化。

如果没有提供TYP格式简历，将整理优化后的内容，严格按照你提供的`resume-ng`模板格式，生成纯Typst格式的`.typ`文件。完全匹配模板的函数和参数，确保后续PDF生成正常。

文件开头必须导入标准的resume-ng模板： ```typst #import "@preview/resume-ng:1.0.0": *```

可参考

`assets\reference\resume.typ`

`assets\typst-pdf-gen\template\main.typ`

## 第四步：生成简历PDF

调用 `python scripts/make_resume.py path/to/resume.typ`

PDF 会生成在当前工作目录。

> 脚本路径相对于 skill base directory，详细说明见上方「脚本路径说明」章节。
