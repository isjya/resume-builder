# 推荐系统工程师 专属简历指引

## 面试官核心关注点
- 对推荐系统全链路的理解（召回、排序、重排）
- 主流推荐模型的实战经验（DeepFM、DIN、SIM等）
- 离线训练与在线推理的工程落地能力
- A/B实验设计与业务指标优化能力
- 冷启动、多样性、公平性等推荐难题的解决经验

## 定制化简历建议
1. **技能部分突出全链路能力**
   - 优先列出：召回/排序模型、离线训练、在线推理、A/B测试
   - 其次列出技术栈：TensorFlow/PyTorch、Spark/Flink、特征工程
   - 最后列出业务指标相关的能力，如CTR/CVR优化、用户留存提升

2. **经历描述突出指标优化**
   - 所有推荐相关项目必须量化业务指标：CTR、CVR、GMV、用户停留时长
   - 详细说明你解决的推荐难题，如冷启动、数据稀疏性、多样性不足
   - 重点描述你对模型的改进和工程落地，如模型压缩、推理性能优化
   - 提及你设计的A/B实验，如何验证模型效果并迭代

3. **结构调整**
   - 推荐相关项目要放在最前面，用最大的篇幅详细描述
   - 技能模块可以前置，快速证明你对推荐技术栈的掌握

## STAR法则示例
- ❌ 错误：负责推荐系统的算法开发
- ✅ 正确：原有推荐系统使用传统协同过滤，存在CTR低、新用户冷启动问题(S)，负责全新深度学习推荐模型的落地(T)，通过调研业界方案，实现了DIN兴趣注意力模型，同时设计了用户冷启动的多源特征融合方案(A)，上线后核心CTR提升67%，新用户首点击转化率提升42%，整体GMV提升18%(R)
